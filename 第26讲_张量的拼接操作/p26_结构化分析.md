## 内容结构分析

### 核心概念
张量拼接的两种方式：`cat` 和 `stack`

### cat函数详解
- **特点**: 不改变维度数
- **规则**: 除了拼接的那个维度外，其他维度数必须保持一致
- **示例**:
  - (2,3) + (2,3) 在dim=0拼接 → (4,3)
  - (2,3) + (2,3) 在dim=1拼接 → (2,6)
  - (2,3) + (4,3) 在dim=0拼接 → (6,3) ✓
  - (2,3) + (4,3) 在dim=1拼接 → 报错（其他维度不一致）

### stack函数详解
- **特点**: 会改变维度数（新增一个维度）
- **规则**: 所有输入张量的形状必须完全相同
- **示例**:
  - (2,3) + (2,3) 在dim=0拼接 → (2,2,3)
  - (2,3) + (2,3) 在dim=1拼接 → (2,2,3)
  - 任意维度不一致都会报错

### cat vs stack 对比
| 特性 | cat | stack |
|------|-----|-------|
| 维度变化 | 不改变维度数 | 新增一个维度 |
| 维度要求 | 拼接维度可不同，其他必须相同 | 所有维度必须完全相同 |
| 使用场景 | 组合数据（类似书上册+下册） | 需要保留原始维度结构 |

### 维度越界问题
- 当指定dim超过张量原有维度时报错
- 示例：二维张量无法在dim=2上拼接
- 错误信息：Dimension out of range
