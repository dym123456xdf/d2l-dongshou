## 内容结构分析

### 核心概念
view函数用于修改张量形状，但**只能处理连续的张量**

### 连续性概念
**连续张量**：张量底层数据在内存中的存储顺序与其在张量中的逻辑顺序一致

**不连续张量**：存储顺序与显示顺序不一致
- 典型例子：经过transpose或permute处理后的张量
- view无法处理不连续张量

### 连续性判断
```python
tensor.iscontiguous()  # 返回True表示连续
```

### 转为连续
```python
tensor.contiguous()  # 返回连续的新张量
```

### view使用流程
1. 判断张量是否连续：iscontiguous()
2. 如果不连续，先调用contiguous()
3. 再使用view修改形状

### 各函数对比总结

| 函数 | 功能 | 限制 |
|------|------|------|
| reshape | 改变形状 | 无（自动处理） |
| transpose | 交换维度 | 只能两个 |
| permute | 交换维度 | 任意多个 |
| view | 改变形状 | 只能处理连续张量 |

### 为什么需要view？
1. **提前暴露错误**：不连续时直接报错，而非隐藏问题
2. **灵活性**：满足不同用户需求
3. **安全性**：帮助开发者尽早发现维度问题

### 应用场景
- CV（计算机视觉）领域常用view
- 特征图处理
- 批量数据维度调整
