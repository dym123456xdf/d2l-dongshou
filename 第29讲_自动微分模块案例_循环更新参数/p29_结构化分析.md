## 内容结构分析

### 核心概念
通过循环迭代实现参数的多轮更新，演示梯度下降的完整过程

### 关键问题：梯度累加
- **问题**: 默认情况下梯度会累加，不会自动清零
- **影响**: 如果不清零，梯度会不断累积（20→40→60...）
- **解决**: 使用 `W.grad.zero_()` 清零梯度

### 梯度清零时机
- 在每次反向传播之前需要清零梯度
- 但第一次反向传播前没有梯度，无需清零
- 需要做非空判断

### 迭代更新流程
1. 定义初始权重 W = 10.0
2. 定义损失函数 loss = W² + 20
3. 循环100次:
   - 正向传播计算loss
   - 清零梯度（非空判断）
   - 反向传播计算梯度
   - 更新权重: W = W - 学习率 × 梯度

### 代码演示
```python
import torch

W = torch.tensor(10.0, requires_grad=True, dtype=torch.float32)
learning_rate = 0.01

print(f"初始权重: {W}")
print(f"初始损失: {W**2 + 20}")  # 120

for i in range(1, 101):
    # 正向传播
    loss = W ** 2 + 20
    
    # 清零梯度（非空判断）
    if W.grad is not None:
        W.grad.zero_()
    
    # 反向传播
    loss.backward()
    
    # 更新权重
    W.data = W.data - learning_rate * W.grad
    
    if i <= 5 or i == 100:
        print(f"第{i}次: W={W.data:.5f}, loss={loss.item():.5f}")

# 最终结果: 权重趋近最优解，损失从120降到21
```

### 手动验证（第一次迭代）
- W旧 = 10
- loss = W² + 20 = 120
- d(loss)/dW = 2W = 20
- W新 = 10 - 0.01 × 20 = 9.8

### 梯度清零原因
- 梯度默认会累加
- 不清零会导致梯度越来越大
- 需要每轮清零后重新计算
